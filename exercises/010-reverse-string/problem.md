# Exercise 010 — 文字列の反転（日本語）

## Objective

- JSON-RPC 2.0 のメソッド `reverse` を実装し、1つの文字列パラメータを受け取り、その文字列を反転して返す。

## Specification

### JSON-RPC フィールド

- `jsonrpc`: 文字列 `"2.0"`。
- `method`: `"reverse"`。
- `params`: 位置引数の配列で文字列1つ、例: `["hello"]`。
- `result`: 反転した文字列、例: `"olleh"`。

### 動作と検証

- パラメータが空文字列の場合、結果は空文字列。
- パラメータが指定されていない、または文字列でない場合は `-32602`（Invalid params）を返す。
- 通知（`id` がないリクエスト）はレスポンスを返さない。
- 実装は Unicode のコードポイントを適切に扱って文字列を反転することが望ましい。

## エラー処理

- 標準の JSON-RPC エラーコードを使用する（`-32700`、`-32600`、`-32601`、`-32602`、`-32603`）。

## 例

リクエスト:
```json
{"jsonrpc":"2.0","method":"reverse","params":["hello"],"id":1}
```

レスポンス:
```json
{"jsonrpc":"2.0","result":"olleh","id":1}
```

## 受け入れ条件

- ASCII および Unicode 文字列を適切に反転し、型エラーには `-32602` を返す。
